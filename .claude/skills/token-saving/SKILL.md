# トークン節約ガイド

このリポジトリで作業する際、不要なトークン消費を避けるための実践ルール。

## 1. 検索は専用ツールで、Bashで生コマンドを叩かない
- ファイル名を探すときは `find` ではなく Glob を使う。
- コード内容を探すときは `grep`/`rg` ではなく Grep を使う。
- `output_mode: "files_with_matches"` や `"count"` でまず絞り込み、本当に中身が必要なファイルだけ `"content"` で見る。
- `head_limit` を使って結果件数を制限する(デフォルトの上限に頼りきらない)。

## 2. ファイルは必要な範囲だけ読む
- 大きいファイルは `Read` の `offset`/`limit` で該当箇所だけ読む。全文読み込みは構造把握が本当に必要なときだけ。
- 一度読んだファイルを読み直さない。Edit/Write は失敗時にエラーを返すので、確認のための再読み込みは不要。
- `cat`/`head`/`tail` をBashで使わず、Readを使う。

## 3. 大規模な調査はサブエージェントに逃がす
- 3回以上の探索的検索が必要そうな調査は、Agent (Explore など) に投げてメインの会話コンテキストを汚さない。
- サブエージェントには「何を知りたいか」を明確に伝え、結果は要約で返してもらう。

## 4. 出力は要約する、生ログを貼らない
- コマンド出力・ビルドログ・diffなどは、関係する部分だけ抜粋するか要約する。
- 大量出力が予想されるコマンドは、事前に `| head` やフィルタ引数で絞ってから実行する。

## 5. ツール呼び出しをまとめる
- 依存関係のない複数の調査・確認は、同じレスポンス内で並列に呼び出し、往復回数を減らす。
- 同じ情報を得るために似たツール呼び出しを何度も繰り返さない。

## 6. 会話・応答も簡潔に
- ユーザーへの返答は結論から。前置きや思考過程の垂れ流しをしない。
- 求められていない説明・ドキュメント・コメントを追加しない。
